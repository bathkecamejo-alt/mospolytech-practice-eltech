## 2. Технический разбор онлайн-сессии с экспертами компании «Яндекс»

В рамках взаимодействия с индустриальным партнёром была подробно изучена архитектурная сессия, проведённая ведущим разработчиком Александром Голубевым (Яндекс Доставка, секция «Под капотом C++»). Тема доклада: *«Когда действительно нужны алгоритмы: опыт оптимизации KD-Tree»*.

### 2.1. Сущность технологии KD-Tree (K-мерные деревья)
В рамках вебинара был разобран практический кейс пространственного ветвления данных для гео-сервисов. Для быстрой обработки запросов поиска ближайших курьеров и точек выдачи стандартные базы данных не подходят из-за вычислительной сложности алгоритма полного перебора $O(N)$. 

Использование структуры **KD-Tree** (K-dimensional tree) позволяет разбить двухмерное пространство (координаты на карте) на изолированные сектора с помощью разделяющих плоскостей. Это снижает среднюю сложность поиска до $O(\log N)$.

### 2.2. Особенности программной реализации алгоритма
На сессии был продемонстрирован работающий листинг оптимизированной функции `Search`, выполняющей обход дерева без использования стандартной тяжеловесной рекурсии, которая опасна переполнением стека (Stack Overflow) в высоконагруженных системах. 

Обход реализован через кастомный стек структур `std::stack<NodePtr>` итеративным методом, что минимизирует накладные расходы на вызов функций на системном уровне.

### 2.3. Анализ производительности на нагрузочном стенде
Важной частью взаимодействия стал разбор метрик производительности, полученных на тестовом стенде Яндекса при обработке массивов данных (от 381 до 21974 записей). Были проанализированы показатели задержек (timings в миллисекундах) для различных перцентилей (100%, 99.9%, 99%, 98%, 95%, 50%). 

Анализ показал, что оптимизация алгоритма фильтрации (`gp_finish_filters_time`) и времени поиска (`search_time`) позволяет удерживать общую задержку обработки запроса (`total_time`) в пределах допустимых SLA даже под экстремальной нагрузкой.

### 2.4. Связь с проектом «Eltech» и Информационной Безопасностью
Полученные знания о пространственных деревьях и алгоритмах фильтрации имеют прямую связь с разрабатываемым приложением «Eltech» для Московского Политеха:
1. **Оптимизация кэширования:** При проектировании SDK локального личного кабинета эффективная фильтрация JSON-баз данных (например, `students.json`) позволяет разгрузить RAM мобильного устройства.
