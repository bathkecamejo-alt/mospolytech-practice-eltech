## 1. Теоретические основы и архитектура системы

Программный комплекс сочетает в себе два фундаментальных механизма защиты информации: криптографическое хэширование данных и симметричное потоковое шифрование методом гаммирования.

### 1.1. Алгоритм хэширования DJB2
Для верификации целостности строк и генерации производных ключей используется модифицированный алгоритм хэширования **DJB2**, разработанный профессором Дэниелом Дж. Бернштейном. Алгоритм относится к классу некриптографических хэш-функций с чрезвычайно низким уровнем коллизий и высокой скоростью работы, что критично для встраиваемых SDK мобильных приложений.

Вычисление хэша основано на битовых сдвигах и добавлении ASCII-значений символов к базовому числу-константе:
$$hash_{i} = (hash_{i-1} \times 33) \oplus c$$

В качестве начального значения (Magic Constant) используется простое число `5381`, которое экспериментально обеспечивает наилучшее распределение хэш-функции в многомерном пространстве и минимизирует лавинный эффект.

### 1.2. Метод потокового шифрования (XOR-гаммирование)
Шифрование файлов основано на принципе симметричного гаммирования. Исходный текст (или бинарный поток) $P$ побайтово складывается по модулю 2 с последовательностью символов ключа (гаммой) $K$:
$$C_i = P_i \oplus K_i$$

Поскольку операция $\oplus$ (XOR) является инволютивной (обратной самой себе), дешифрование выполняется абсолютно тем же алгоритмом:
$$P_i = C_i \oplus K_i$$

**Модификация (Творческая часть):** Классический шифр XOR уязвим к частотному анализу, если ключ статичен и короче шифруемого текста. Чтобы обойти это ограничение, в модуль внедрён **механизм динамического соления**. Пользовательский ключ предварительно смешивается со статической солью, пропускается через алгоритм DJB2, и шифрование выполняется уже динамической псевдослучайной последовательностью, что скрывает структуру исходного файла.

## 2. Программная реализация исходного кода

Исходный код утилиты декомпозирован на логические блоки: вычисление хэш-суммы, бинарная потоковая обработка файлов и консольный интерфейс управления (CLI).

```cpp
#include <iostream>
#include <fstream>
#include <string>
#include <vector>

// Реализация хэш-функции DJB2 для работы с солью
std::string calculateHash(const std::string& str) {
    unsigned long hash = 5381;
    for (char c : str) {
        // Вычисление сдвига: hash * 33 + c
        hash = ((hash << 5) + hash) + c;
    }
    return std::to_string(hash);
}

// Потоковое шифрование/дешифрование в бинарном режиме
bool processFile(const std::string& inputPath, const std::string& outputPath, const std::string& key) {
    std::ifstream inFile(inputPath, std::ios::binary);
    std::ofstream outFile(outputPath, std::ios::binary);

    if (!inFile.is_open() || !outFile.is_open()) {
        return false;
    }

    // Генерация динамической гаммы на основе хэша ключа
    std::string dynamicKey = calculateHash(key);
    size_t keyIndex = 0;

    char byte;
    // Побайтовое чтение потока до конца файла
    while (inFile.get(byte)) {
        // Операция XOR текущего байта с символом псевдослучайной гаммы
        char processedByte = byte ^ dynamicKey[keyIndex % dynamicKey.length()];
        outFile.put(processedByte);
        keyIndex++;
    }

    inFile.close();
    outFile.close();
    return true;
}
